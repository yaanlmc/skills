# 工资表-人员核对

## 适用场景

用户发来工资表 Excel，要求核对：
- 汇总表 vs 分表人数是否一致
- 哪些人在汇总表但不在分表（漏填）
- 哪些人在分表但不在汇总表（多填）

## 输入

用户提供的工资表文件（.xlsx / .xls），可能需要密码解密（密码通常由用户提供）。

## 核对流程

### Step 1：解密（如需）

如果文件是加密的，用 `msoffcrypto-tool` 解密：

```powershell
msoffcrypto-tool encrypted.xlsx decrypted.xlsx -d -p <密码>
```

### Step 2：读取并分析

运行核对脚本（见下方 "使用脚本"），或按以下手动逻辑分析：

1. **汇总表** → 取姓名列（C列，通常第3列），从第4行起提取所有人名
2. **各分表** → 找到姓名列（遍历前几列找含"姓名"的列），提取人名
3. **集合运算**：汇总 - 分表（只在汇总）| 分表 - 汇总（只在分表）
4. **差异分类**：根据姓名列旁边是否有学部/部门列，输出每人的部门信息

### Step 3：解读结果

常见差异原因：
- 后勤公司人员（如司机、维修、办公室）不在6学部分表中，属正常设计
- 离职/入职人员只出现在一侧
- 汇总表含"合计行"（如"分表实发合计"）需排除

## 使用脚本

运行 `scripts/check_headcount.py`：

```powershell
python scripts/check_headcount.py <Excel文件路径> [--sheets "表1;表2;表3"]
```

- `--sheets`：指定要核对的分表名（用分号分隔），默认自动识别最后几个分表
- 输出：各表人数、只在汇总的人、只在分表的人、每人所在部门

## 脚本逻辑说明

- 姓名列识别：遍历前8列，查找表头含"姓名"的列
- 排除非姓名项：序号、小计行、合计行、空白行
- 部门识别：汇总表中的学部（C9）和部门（C10）列
